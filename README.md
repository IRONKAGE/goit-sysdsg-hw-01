# **Завдання 1: Проектування розподіленої системи кешування для онлайн-платформи**

## **Опис завдання:**

Ви працюєте архітектором у компанії, яка розробляє популярну онлайн-платформу для перегляду відеоконтенту (подібну до *YouTube*). Платформа швидко зростає, і ви зіткнулися з проблемами продуктивності та масштабованості. Вам необхідно розробити комплексне рішення для оптимізації системи з використанням розподілених даних та кешування.

## **Вимоги до системи:**

1. Платформа має обслуговувати мільйони користувачів по всьому світу.
2. Користувачі можуть переглядати, завантажувати та коментувати відео.
3. Система повинна підтримувати рекомендації відео на основі переглядів користувача.
4. Необхідно забезпечити високу доступність та низьку затримку при доступі до контенту.
5. Система має ефективно масштабуватися при збільшенні навантаження.

**Припущення:**

- Активних користувачів: 100 мільйонів щомісяця
- Середня кількість переглядів відео на користувача: 5 на день
- Середня тривалість відео: 5 хвилин
- Середня якість відео: 5 Мбіт/с (HD якість)
- Нових відео завантажується: 500 000 на день
- Середній розмір метаданих відео: 2 КБ
- Середній розмір коментаря: 100 байтів
- Середня кількість коментарів на відео: 10

## **Завдання:**

### **Частина 1: Аналіз вимог та оцінка масштабу:**

1. Використовуючи **Back-of-the-Envelope розрахунки**, оцініть:
   - Очікуваний обсяг даних для зберігання (відео, метадані, коментарі)
   - Пропускну здатність мережі для обслуговування користувачів
   - Необхідні обчислювальні ресурси
2. Проаналізуйте, які нефункціональні вимоги є найбільш критичними для системи, використовуючи принципи **CAP-теореми**. Розгорнуто обґрунтуйте свій вибір.

---

### **Частина 2: Проектування моделі даних:**

1. Розробіть **схеми даних** для основних сутностей системи:
   - Користувачі
   - Відео (зокрема метадані та самі відеофайли)
   - Коментарі
   - Історія переглядів
   - Рекомендації
2. Для кожної сутності **визначте**:
   - Яку модель даних використати (реляційну, документо-орієнтовану, графову тощо)
   - Обґрунтуйте свій вибір, враховуючи характер даних та операцій з ними
   - Опишіть структуру даних (таблиці, колекції, відношення)
3. Визначте, які дані потребують сильної узгодженості, а які можуть використовувати **eventual consistency**.

---

### **Частина 3: Розподілення даних:**

1. Розробіть **стратегію реплікації** для різних типів даних:
   - Яку модель реплікації (single-leader, multi-leader, leaderless) використати для кожного типу даних
   - Обґрунтуйте свій вибір
2. Спроектуйте **стратегію секціонування (шардингу)** для масштабування бази даних:
   - Який підхід до секціонування використати (за хешем, за інтервалами, випадковий)
   - Як розв’язати проблему hot spots
   - Як організувати секціонування вторинних індексів
3. Опишіть, як ваша система буде **обробляти**:
   - Відмову реплік
   - Відмову лідера (якщо використовується leader-based реплікація)
   - Replication lag та його вплив на користувацький досвід

---

### **Частина 4: Стратегія кешування:**

1. Розробіть комплексну **стратегію кешування** для системи:
   - Визначте, які дані потрібно кешувати на різних рівнях (клієнт, CDN, сервер, база даних)
   - Обґрунтуйте вибір стратегій кешування (Cache Aside, Read Through, Write Through, Write Back, Write Around) для різних типів даних
2. Спроектуйте **розподілений кеш** для метаданих відео та статистики переглядів:
   - Виберіть між виділеним та розміщеним кластером кешу
   - Опишіть стратегію заміщення кешу (LRU, LFU, FIFO тощо)
   - Запропонуйте механізми для забезпечення узгодженості кешу
3. Розробіть **стратегію використання CDN** для доставки відеоконтенту:
   - Визначте, який підхід використати (Pull CDN чи Push CDN)
   - Опишіть, як система буде обробляти оновлення контенту в CDN
   - Запропонуйте рішення для оптимізації доставки контенту користувачам з різних географічних регіонів

---

### **Частина 5: Інтеграція та балансування:**

1. Створіть **високорівневу архітектурну діаграму системи**, що показує взаємодію всіх компонентів:
   - Бази даних та їхні репліки
   - Кеш-сервери
   - CDN
   - Сервіси застосунку
2. Опишіть, як **система забезпечує баланс** між:
   - Продуктивністю та узгодженістю даних
   - Доступністю та розділенням мережі (відповідно до CAP-теореми)

---
